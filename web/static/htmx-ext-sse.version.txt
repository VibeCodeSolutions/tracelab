Why a separate file
-------------------

htmx v2.0.4 (the core vendored as htmx.min.js) does **not** bundle
the Server-Sent Events extension. The htmx project ships extensions
as separate packages (htmx-ext-sse, htmx-ext-ws, …) so consumers
opt-in by loading the script alongside htmx core and adding
hx-ext="sse" to the subscribing element.

Tracelab needs SSE for the live-tail tab (ADR-012 Decision Block:
Option A — SSE on /dashboard/stream). Loading the SSE extension here
keeps the dashboard auth posture (permanently Loopback-only,
ADR-011 Consequences) intact: no CDN fetch, no privacy beacon,
offline-build reproducible, single-binary distribution preserved.

Upgrade procedure
-----------------

A version bump is an explicit commit touching this file plus
htmx-ext-sse.js. Steps:

  curl -sSL -o web/static/htmx-ext-sse.js \
      https://unpkg.com/htmx-ext-sse@<NEW>/sse.js
  sha256sum web/static/htmx-ext-sse.js

Update Version / Source / SHA256 / Bytes / Pinned in this file. The
dashboard SSE smoke tests must still pass:

  go test -race -count=1 ./internal/dashboard/...
